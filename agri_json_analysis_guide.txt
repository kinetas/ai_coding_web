1. 먼저 결론

이 데이터는 unit_sz(수량)와 unit(단위)이 서로 달라서, exmn_dd_avg_prc(원시 가격)만으로는 품목 간 비교 통계를 바로 내면 왜곡될 가능성이 큽니다.

분석 실무 기준으로는 아래처럼 2트랙으로 나누는 것이 가장 안전합니다.

- 같은 포장/같은 단위 비교: exmn_dd_avg_prc 사용
- 서로 다른 중량 단위 비교: exmn_dd_cnvs_avg_prc 사용

핵심은 "무조건 item 기준으로 합치지 말고", 어떤 통계인지에 따라 키를 다르게 잡는 것입니다.


2. 필드 역할 해석

- ctgry_nm: 대카테고리
- exmn_dd_avg_prc: 조사일의 실제 거래 단위 가격
- exmn_dd_cnvs_avg_prc: 환산 가격
- exmn_ymd: 조사일
- grd_cd, grd_nm: 등급
- item_cd, item_nm: 품목
- se_cd, se_nm: 거래 단계/시장 구분(예: 중도매, 소매)
- unit: 단위(kg, g, 개, 마리 등)
- unit_sz: 수량/규격 크기
- vrty_cd, vrty_nm: 품종/규격명
- ww1_bfr_* ~ ww4_bfr_*: 1주~4주 전 가격

3-4. 환산가격(exmn_dd_cnvs_avg_prc)의 의미
- kg, g 계열은 원시가격과 환산가격이 다르게 들어오는 경우가 많음
- 개, 마리, 포기, 구, 장, 묶음, 손, 속, 봉, L 계열은 대부분 원시가격과 환산가격이 동일함

즉, 실무적으로 보면 exmn_dd_cnvs_avg_prc는 "중량 계열(kg/g) 비교용 환산 단가"로 보는 것이 맞고,
개수/마리/포기 같은 비중량 단위는 환산이 사실상 안 된 상태라고 보는 것이 안전합니다.

예시
- 쌀 20kg: avg 47,225 / cnvs 2,361 -> 사실상 kg당 가격
- 감자 100g: avg 443 / cnvs 4,429 -> 사실상 kg당 가격
- 수박 1개: avg 19,350 / cnvs 19,350 -> 개당 가격 그대로
- 오이 10개: avg 18,618 / cnvs 18,618 -> 묶음 가격 그대로


4. 어떤 키로 통계를 내야 하나

4-1. 가장 안전한 "원시 시계열 키"

아래 키를 하나의 시계열 단위로 보면 가장 안전합니다.

- item_cd
- vrty_cd
- grd_cd
- se_cd
- unit
- unit_sz

권장 표시용 보조 컬럼
- item_nm
- vrty_nm
- grd_nm
- se_nm
- ctgry_nm

권장 시계열 식별키 예시
- item_cd + vrty_cd + grd_cd + se_cd + unit + unit_sz

이 키는 "같은 품목, 같은 품종, 같은 등급, 같은 거래단계, 같은 포장규격"을 뜻하므로
exmn_dd_avg_prc를 그대로 써도 비교가 깨지지 않습니다.

4-2. 환산 단가 기반 비교 키

중량 계열(kg/g)만 따로 묶어서 아래 키로 비교할 수 있습니다.

- item_cd
- vrty_cd
- grd_cd
- se_cd

여기에 exmn_dd_cnvs_avg_prc를 사용하면,
20kg 쌀과 4kg 쌀 같은 서로 다른 포장규격도 같은 축에서 비교할 수 있습니다.

단, 이 방식은 unit이 kg/g처럼 환산 가능한 경우에만 권장합니다.

4-3. 절대 바로 합치면 안 되는 경우

아래는 같은 item_nm이라도 바로 평균 내면 안 됩니다.

- 품종이 다른 경우: vrty_cd, vrty_nm
- 등급이 다른 경우: grd_cd, grd_nm
- 시장 단계가 다른 경우: se_cd, se_nm
- 비중량 단위와 중량 단위를 섞는 경우
- 포장 규격(unit_sz)이 다른데 exmn_dd_avg_prc를 그대로 평균내는 경우


5. 분석 목적별 추천 통계 방법

5-1. 가격 추이 분석

목적
- 특정 상품이 최근 오르는지 내리는지 보기

추천 키
- 원시 가격 기준: item_cd + vrty_cd + grd_cd + se_cd + unit + unit_sz
- 환산 가격 기준: item_cd + vrty_cd + grd_cd + se_cd

추천 지표
- 일자별 평균가격
- 전주 대비 증감률
- 4주 대비 증감률
- 이동평균(3기, 5기)
- 변동성(표준편차, CV)

5-2. 품목 간 비교

목적
- 감자 vs 고구마처럼 품목 간 상대 가격 비교

추천 키
- 중량 환산 가능 품목만 대상으로
- item_cd + se_cd + grd_cd 기준

추천 값
- exmn_dd_cnvs_avg_prc

주의
- 개, 마리, 포기 단위는 같은 축 비교에서 제외하거나 별도 그룹으로 분리

5-3. 등급별 프리미엄 분석

목적
- 상품/중품 가격 차이 보기

추천 키
- item_cd + vrty_cd + se_cd + unit + unit_sz

추천 지표
- 상품 대비 중품 프리미엄
- 등급별 가격지수

5-4. 소매 vs 중도매 차이

목적
- 유통단계별 마진/격차 보기

추천 키
- item_cd + vrty_cd + grd_cd + unit + unit_sz

분석 방법
- se_cd 또는 se_nm 기준으로 그룹화
- 같은 날짜에 단계별 가격 차이 계산

5-5. 품종별 차이

목적
- 같은 품목 안에서 품종 프리미엄 보기

추천 키
- item_cd + grd_cd + se_cd + unit + unit_sz

분석 방법
- vrty_cd 기준 비교
- 환산 가능 품목은 cnvs 가격 기준으로 비교

5-6. 카테고리 지수

목적
- 채소류 전체, 과일류 전체가 오르는지 보기

추천 방법
- 먼저 하위 시계열을 표준화
- 각 시계열을 기준시점=100 지수로 변환
- 이후 ctgry_nm 단위로 평균

주의
- 원 단위 가격을 단순 평균하지 말고, 먼저 지수화한 뒤 평균


6. unit_sz와 unit이 다를 때 실무 규칙

규칙 1
- unit이 kg 또는 g 이면 exmn_dd_cnvs_avg_prc를 우선 사용

규칙 2
- unit이 개, 마리, 포기, 구, 장, 묶음, 손, 속, 봉, L 이면 exmn_dd_avg_prc를 사용
- 이 경우 unit_sz를 반드시 키에 포함

규칙 3
- 특수 단위(예: kg(그물망 3포기))는 일반 kg와 합치지 말고 별도 단위군으로 관리

규칙 4
- item_nm이 같아도 vrty_cd, grd_cd, se_cd가 다르면 별도 시계열로 본다

규칙 5
- 전체 시장 대표값이 필요하면, 먼저 세부 시계열별 지수를 만든 뒤 평균한다
- 서로 다른 포장 가격을 원화 그대로 평균내는 방식은 피한다

10. 최종 권고

가장 중요한 기준은 아래 한 줄입니다.

"원시가격(exmn_dd_avg_prc)은 같은 규격끼리만 비교하고, 서로 다른 중량 규격 비교는 환산가격(exmn_dd_cnvs_avg_prc)로 하며, 개수형 단위는 unit_sz/unit을 반드시 유지한다."

즉,
- 규격 그대로 보고 싶으면: unit_sz + unit 포함
- 중량 기준으로 비교하고 싶으면: cnvs 가격 사용
- item_nm 하나만으로 합치지는 말고 item_cd, vrty_cd, grd_cd, se_cd를 함께 본다
